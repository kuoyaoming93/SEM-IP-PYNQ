    \subsection{ICAP and PCAP}\label{icap-and-pcap}

    During boot of the Zynq-7000 Processing System (PS), access to the
configuration logic in the device is given to the PS through the
processor configuration access port (PCAP). This provides a path for the
PS bootloader to download a bitstream to the Zynq-7000 Programmable
Logic (PL). When the PS bootloader completes, the PS and PCAP remain in
control of the configuration logic to support partial reconfiguration of
the PL by the PS.

However, while the PS and PCAP are in control of the configuration
logic, the PL and ICAP are locked out of the configuration logic. In
order for the controller to function, configuration logic access must be
transferred to the ICAP. This is accomplished by clearing PCAP\_PR (bit
27) in the PS device configuration control register (DEVCFG CTRL,
address 0xF8007000).
